# 重新打包 App 技能

## 技能说明

当用户要求重新打包、构建、生成 .exe 安装文件时，激活此技能。将 Python 源码通过 PyInstaller 打包为单个 Windows 可执行文件。

---

## 触发条件

用户说了以下任何一句话时，使用此技能：
- "重新打包"
- "打包 app"
- "生成 exe"
- "构建程序"
- "rebuild"
- "build app"
- "给我一个新包"
- 任何明确表达"要重新生成可执行文件"的请求

---

## 项目打包信息

```
打包工具：PyInstaller（单个文件模式 --onefile）
入口脚本：src/main.py
打包命令：python build.py
输出文件：dist/张家港房价K线图.exe
依赖文件：requirements.txt
```

---

## 执行步骤

按以下顺序一步一步执行，每步都要向用户报告结果：

### 第 1 步：检查 PyInstaller 是否已安装

```bash
pip show pyinstaller
```

- 如果已安装 → 告诉用户版本号，继续下一步
- 如果未安装 → 执行 `pip install pyinstaller`，再继续

### 第 2 步：检查入口文件是否存在

确认 `src/main.py` 文件存在。如果不存在，立即告知用户并停止。

### 第 3 步：执行打包

在项目根目录（`d:/zzz/张家港房价app/`）下运行：

```bash
python build.py
```

- 等待命令执行完毕（可能需要 1-3 分钟）
- 向用户实时报告打包进度和输出日志

### 第 4 步：验证结果

检查 `dist/张家港房价K线图.exe` 是否存在：

- ✅ 存在 → 告诉用户文件路径和文件大小
- ❌ 不存在 → 告诉用户打包失败，并展示错误日志供排查

---

## 注意事项

1. **不要修改 `build.py` 的内容**，直接运行它即可
2. 打包过程会清理 `build/` 和 `dist/` 目录，这是正常行为
3. 打包耗时通常 1-3 分钟，期间不要中断命令
4. 如果打包失败，重点关注 PyInstaller 的报错信息，常见原因：
   - 某个依赖包没装 → 执行 `pip install -r requirements.txt`
   - 杀毒软件拦截 → 临时关闭杀毒后重试
5. 生成的 .exe 文件可以用压缩软件减小体积，但本技能只负责打包，不负责压缩
